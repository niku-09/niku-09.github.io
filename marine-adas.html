Marine ADAS Simulator

A real-time maritime vessel simulator running five simultaneous robotics algorithms ” built from scratch in Python.

Live simulator view: tactical navigation panel (left), PPI radar, SLAM occupancy map, sensor status, and EKF performance monitor.

A from-scratch maritime Autonomous Driving Assistance System (ADAS) simulator in a single ~1800-line Python file — no game engine, no ROS, no external simulation framework. It places a vessel in a 30×30 m harbour filled with rocks, wrecks, buoys, and three moving AIS vessels, then runs five distinct robotics algorithms simultaneously in real time.

Every algorithm is physically grounded: sensor noise models follow real-world MEMS and GPS specifications, the EKF uses the Joseph-form covariance update for long-run numerical stability, and COLREGS Rules 13–16 drive the autopilot’s avoidance response. The goal was to build the full pipeline — raw noisy sensors → Kalman fusion → occupancy mapping → trajectory planning → collision law enforcement — in one coherent, observable system.

Five algorithms, one system
EKF

Extended Kalman Filter · Sensor Fusion

Fuses GPS and IMU streams into a single optimal state estimate (position, heading, speed). Uses the Joseph-form covariance update for numerical stability, and tracks filter health via the NIS Ï‡²(2) statistic in real time.
DWA

Dynamic Window Approach · Obstacle Avoidance

Every 3 frames, samples 35 candidate (speed, turn-rate) pairs within the kinematically reachable window, simulates each 1.5 s forward, discards collisions, and picks the best safe trajectory using a heading + speed + clearance score.
SLAM

Occupancy Grid Mapping

Builds a live 30×30 cell map from 360° LiDAR beam returns using log-odds updates and Bresenham ray traversal — the same architecture as ROS gmapping. The map is built from the EKF pose estimate, not ground truth.
CPA / TCPA

Closest Point of Approach · Collision Prediction

Computes exact DCPA and TCPA for each AIS contact using the closed-form ARPA formula. Fires red/amber/green threat alerts and plots predicted closest-approach markers on the navigation view.
COLREGS

Rules 13“16 · Maritime Collision Avoidance Law

Classifies each vessel encounter as HEAD-ON, GIVE WAY, STAND-ON, or OVERTAKING per the International Regulations for Preventing Collisions at Sea (1972). In autopilot mode, a GIVE WAY contact biases the heading command âˆ’23° to starboard, implementing Rule 15's requirement to yield to vessels on the starboard side.

GPS fires every 20 frames with multipath events (4% probability, 5× noise spike) and 8% packet loss. IMU accumulates gyro drift over time — separating EKF trajectory from ground truth during GPS outages.
