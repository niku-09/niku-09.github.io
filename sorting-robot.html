A low-cost, functional robotic sorting system that autonomously detects, classifies, and places small cardboard packages into designated drop zones ” built to demonstrate last-metre warehouse automation at minimal hardware cost. The system integrates a custom 4-DOF robotic arm, an overhead USB camera, and a Python-based software stack orchestrated through a real-time web dashboard.

Detection Pipeline

An overhead webcam captures the conveyor surface. OpenCV's background subtraction pipeline isolates packages using CLAHE-enhanced frames and Otsu thresholding. Detections are filtered by solidity, minimum area, and a temporal persistence check ” ensuring only stable, real boxes trigger the sorting sequence. An arm exclusion zone masks out the robot itself from the detection frame.

â‘¡ Colour

Colour Classification

Each detected box carries a coloured sticker indicating its routing category: FRAGILE (red), IMMEDIATE (green), ZONE_A (yellow-orange), or DAMAGED (purple). A dual-space classifier ” combining HSV hue ranges with LAB colour signatures ” handles real-world printing variations and ambiguous hues. An inner-crop strategy and shadow-pixel exclusion sharpen accuracy under variable lighting.

Camera-to-Robot Coordinate Transform

Detected pixel coordinates are mapped to robot workspace coordinates via an empirically calibrated rigid transform (rotation + translation), derived using a checkerboard calibration board. A sign-convention correction aligns the camera's Y-axis with the robot's frame, and fixed XY offsets compensate for systematic mount error.

Inverse Kinematics & Motion Planning

A closed-form geometric IK solver (atan2 for base azimuth; law of cosines for shoulder/elbow) computes joint angles for any target pose. The elbow-up configuration is preferred. FK verification confirms solutions within Â±0.5 cm before dispatch. Servo commands are sequenced with a staged lift strategy to manage torque on the forearm joint.

Pick, Sort & Place

The arm moves to the detected box position, closes the gripper, lifts in stages, rotates to the pre-mapped drop-zone coordinate for that label, and releases. Drop zones are fixed in robot coordinates: FRAGILE, IMMEDIATE, ZONE_A, and DAMAGED bins arranged across the workspace.

SortOS Dashboard

A Flask-based web dashboard (SortOS) provides a live MJPEG feed with detection overlays, a running inventory log, confidence threshold slider, sort-event toasts, and a trend chart ” giving operators full visibility without needing to touch the terminal.
